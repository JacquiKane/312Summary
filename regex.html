Topic: Regular Expressions

A regular expression is a set of symbols that describes a text pattern. A regular expression can be used to ensure correct formatting. It does not mean that the data is accurate, just that it has the required number of digits and letters, and that the user has entered data in accordance with expected data format. Regular Expressions are used for matching, searching, updating text, on the basis of mirroring a set pattern. Programming languages (like JavaScript, Python...) use regular expressions to validate data input. The term Regular Expression, is often abbreviated to Regex.

Matching Literals

A literal expression is simply a set of characters, like a word or a partial word. Use / to start delineating the regular expression, and / to finish it. Any characters in-between / and / form the regular expression. Below, 3 characters, ory, are matched.

Using Character Ranges

Character ranges help with specifying a series of characters in a range, like a through z, or 1 through 9 (digits are characters, with numeric meaning). Characters that can be matched in this way should logically follow each other. Character ranges are specified with a - (dash) meta-character, and the character range feature assumes that the characters have an ordering.

Using The Negate Meta-character

Characters are dis-included with the negate meta-character, used inside a character set. For example, suppose we want to target NOT H, followed by ea, followed by any character, below.
